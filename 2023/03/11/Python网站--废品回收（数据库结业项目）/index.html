需求分析

功能性需求分析

根据需求调研，前台用户分为客户、商户、取货员，功能设计包括下单、评价、接单、订单确认、取货确认、个人信息查看，修改个人信息，实名认证，登录，注册等功能，客户用户的功能设计包括废品下单、订单详情等。商户的功能设计包括对客户的单进行接单、价格确认、将订单发送给取货员、对取货员返回的订单进行确认，取货员的功能设计包括对商户派的单进行接单、取货送货确认，管理员用户的功能包括审核信息、用户及订单的增、删、改、查、回收种类的管理等。

表结构设计

用户登录需要账户密码，下单时需要填写姓名、联系电话及地址便于商户与取货员查看订单信息取货回收，注册时需要有 id 账号方便管理员管理。用户信息表结构如表

商户登录需要账户密码，接单时需要填写姓名、联系电话及地址便于取货员查看信息送货，注册时需要有 id 账号方便管理员管理。商户表结构如表 4.2 所示。
