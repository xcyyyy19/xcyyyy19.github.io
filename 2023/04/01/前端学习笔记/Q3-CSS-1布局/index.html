    易错点：

    1. display：flex；

      常用于有高度的类（body也能用）。且不能用于有其它display需求的类。（如inline-block）。

    2. 主轴侧轴的改变

      当flex-derection的值改变时，两个轴也会跟着变，也就是说justify-content和align-items也要变。

    3. flex的作用域

      只有所在的类，所在类的子类都不算。比如body里写了的flex，justify只能控制body的布局，想控制内部元素的布局就只能嵌套，在.div里也写flex。

    4. 什么时候改点，什么时候改div？

    知识点：

    1.布局原理：采用 Flex 布局的元素，称为容器。它的所有子元素自动成为容器成员，称为项目。

    2.易错点：父盒子设为 flex 后，子元素的float、clear和 vertical-align属性将失效。

    3.flex布局父项常见属性
    flex-direction：设置主轴的方

    ​ row：行 column：柱子

    ​ flex-wrap：设置子元素是否换行

    ​ nowrap 不换行 (装不下会缩小子元素的宽度)
    ​ wrap 换行

    ​ justify-content：设置主轴上的子元素排列方式【【【【【不分单行多行吗？】】】】】】

    ​ align-items：设置侧轴上的子元素排列方式（单行）

    ​ 该属性是控制子项在侧轴上的排列方式 在子项为单行的时候使用
    ​ flex-start 从头部开始
    ​ flex-end 从尾部开始
    ​ center 居中显示
    ​ stretch 拉伸(不给宽度/高度)

    ​ align-content： 设置侧轴上的子元素的排列方式（多行）

    ​

    ​ align-content 和align-items区别
    ​ align-items 适用于单行情况下， 只有上对齐、下对齐、居中和 拉伸
    ​ align-content适用多行换行的情况（单行无效）

    ​ flex-flow：复合属性，相当于同时设置了 flex-direction 和 flex-wrap

    3.flex布局子项常见属性【少用】
    flex子项目占的份数
    align-self控制子项自己在侧轴的排列方式
    order属性定义子项的排列顺序（前后顺序）
    4.flex属性
    flex 属性定义子项目分配剩余空间，用flex来表示占多少份数。

    ​ 【重点：flex：1；表示flex-grow1；&& flex-shrink：1；

    ​ 意味着如果盒子剩余的布局大就自适应缩小，反之就自适应增大】

    5.align-self控制子项自己在侧轴上的排列方式【少用】
    align-self 属性允许单个项目有与其他项目不一样的对齐方式，可覆盖 align-items 属性。
    默认值为 auto，表示继承父元素的 align-items 属性，如果没有父元素，则等同于 stretch。

2.定位

1.position属性

CSS 布局 - position 属性 (w3school.com.cn)

​ 概念：

​ position 属性规定应用于元素的定位方法的类型。

​ 五个不同的位置值：

​ static：正常（默认）。

​ relative（相对定位）：改变relation所在的元素，在该元素的父元素中的相对位置。

​ 易错点：如果附近有块级元素，它的相对定位会被影响。

​ absolute（绝对定位）：

​ fixed（固定定位）：

​ sticky（粘性定位）：根据用户的滚动位置进行定位。

​ 粘性元素根据滚动位置在相对（relative）和固定（fixed）之间切换。

​ 起先它会被相对定位，直到在视口中遇到给定的偏移位置为止

​ 然后将其“粘贴”在适当的位置（比如 position:fixed）。

​ index属性

​ 指定元素的堆栈顺序（哪个元素应放置在其他元素的前面或后面，正负都可）。

outline和border的区别

​ 主要区别在于:

​ 1.应用场景:一般border能适用于任何元素的样式，outline一般用于表单组件(比如input/textarea)

​ 2.特性不一样：border一般会被计算在元素的宽高大小里面，outline不会

相关层次图

margin：外边距

border：边框

outline：边框

padding：内边距

border和outline的区别：

1.border占用空间，outline不占用空间，不会影响元素的尺寸和位置。

2.border 可应用于几乎所有有形的html元素，而 outline 是针对链接、表单控件和ImageMap等元素设计。从而另一个区别也可以推理出，那就是： outline 的效果将随元素的 focus 而自动出现，相应的由 blur 而自动消失。这些都是浏览器的默认行为，无需JavaScript配合CSS来控制。

利用这些特性，我们常用outline来设置悬停的动态效果。之前一直困扰我的问题：hover加边框会导致元素定位和大小的改变，同时也会影响其他元素位置的改变，现在终于解决了。
